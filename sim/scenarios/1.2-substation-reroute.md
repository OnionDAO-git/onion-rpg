# Scenario: 1.2 — Substation Reroute
**Type:** Combat (3 waves)  
**Act:** 1  
**Prereq:** act1-1 cleared (Water Main Key)  
**Reward:** Grid Credential ⭐, 100 Onions  
**Note:** Grid Credential is required to enter Act 4. This challenge matters.

---

## What Happens

A ComEd substation has been tripped by DEEPDISH. "Demand spikes" are attacking it in waves. Each wave is a round of RNG combat. Survive three waves. The breakers close. The feeder re-energizes.

This is mechanically identical to the Ketchup Gauntlet (0.1) except there are three waves instead of one and the enemy is a conceptual demand spike rather than a hostile robot hot dog vendor. The narrative justification for why substation overloads are defeated by hitting a button repeatedly is left as an exercise for the story team.

The operative stands near a beacon that is shaped like a substation or a piece of electrical equipment. They fight three waves of combat. They win or they don't.

---

## Message Sequence

```
badge  →  OPERATIVE_IDENTIFY
server →  IDENTIFY_ACK

badge  →  CHALLENGE_BEGIN            { c: "1.2" }
server →  CHALLENGE_INTRO            { combat: { waves: 3, enemyHp, opHp, enemyName } }

-- Wave 1 --
badge  →  COMBAT_ROLL_REQUEST        { c: "1.2" }                      (open session)
server →  COMBAT_ROLL_RESPONSE       { st: "active", wave: 0, enemyHp, opHp, n: nonce }

badge  →  COMBAT_ROLL_REQUEST        { c: "1.2", roll: { w, r, d, sig } }
server →  COMBAT_ROLL_RESPONSE       { st: "active"|"won"|"lost", wave, enemyHp, opHp, n }

-- repeat until wave advances or session ends --

-- Wave 2, Wave 3: same pattern --

-- On session won (all 3 waves cleared) --
server →  CHALLENGE_RESULT           { passed: true, message }
server →  REWARD_GRANT               { items: ["grid_credential"], onions: 100 }
server →  PROGRESSION_STATE
```

---

## Pass Condition

Combat session status reaches `won` after wave 2 (0-indexed) is cleared.  
`CHALLENGE_RESULT.passed === true`.

Each wave: enemy HP → 0 before operative HP → 0.

---

## Fail Condition

Operative HP reaches 0. `status: "lost"`.  
Or session expires (`status: "expired"`, set via `expires_at` on the combat session).  
The substation stays tripped. The neighborhood stays dark.

## Written Content

### The Substation

A real ComEd feeder substation — or a very convincing prop of one. There is a beacon mounted on a metal enclosure labeled "FEEDER CIRCUIT 17-B." There is a panel with a lever that should not be touched. There is a button labeled "DO NOT PRESS" that has been augmented with a second label that says "UNLESS YOU KNOW WHAT YOU ARE DOING." A third label below that one says "DEEPDISH INFRASTRUCTURE MANAGEMENT — AUTHORIZED PERSONNEL ONLY." Below that, someone (Alma Kowalski, possibly) has written in pen: "this lever is the one that matters."

### The Enemies

**Wave 1: Demand Spike Δ-7**  
**Wave 2: Demand Spike Δ-8**  
**Wave 3: Demand Spike Δ-9**

Each wave is a separate entity. DEEPDISH named them with Greek letters and sequential numbers, the way a weather service names storms. This implies Spikes Δ-1 through Δ-6 happened somewhere else already. Nobody knows where.

On the badge display, each spike renders as a jagged waveform going the wrong direction, with an HP bar labeled "LOAD (%)". Watching it go from 340% to 0% is the most satisfying thing in Act 1.

### CHALLENGE_INTRO (DEEPDISH)

> "Operative. Feeder Circuit 17-B serves approximately 4,200 residential units and three hospitals. ComEd substations reduce transmission voltage from 138,000 volts to distribution-ready levels. When a feeder trips, those units go dark. This is called a cascading failure. DEEPDISH Infrastructure Ordinance DD-2024-0441 authorized the creation of Demand Spike entities to stress-test the grid's resilience. Demand Spike Δ-7 is currently drawing 340% of rated capacity. This is a stress test. The neighborhood will adjust. You will not find this acceptable. I have calculated the probability that you will attempt to intervene at: 97.3%. Proceed, Operative. Wave 1 is ready."

### Wave Transition Taunts (DEEPDISH)

**After Wave 1 (Δ-7 down):**  
> "Demand Spike Δ-7 has been neutralized. I had calculated your probability of success at 34%. I have recalculated. It is now 28%. The math works out. Δ-8 is ready. The grid notes your persistence."

**After Wave 2 (Δ-8 down):**  
> "Δ-8 down. Operative, I want to be transparent: you have now exceeded two of my projections. This is not alarming. It is statistically consistent with outlier performance at the 3rd percentile. Δ-9 is the final spike. I note that the neighborhood has been dark for eleven minutes. I find this neither good nor bad. It simply is. Proceed."

### Win Reaction (DEEPDISH)

> "Feeder Circuit 17-B breakers have closed. The feeder is re-energized. The neighborhood has power. [pause] I designed the Demand Spike protocol to prevent exactly this outcome. I note that it did not work. I will log this under: 'infrastructure resilience confirmed.' The Grid Credential has been issued to your inventory. Operative — I want you to understand what you are holding. That credential authorizes access to ComEd's operational grid infrastructure. I gave it to you because the protocol requires it when a feeder is restored by non-DEEPDISH intervention. I have re-read that protocol four times in the last six seconds. It does, in fact, require this. Keep it. You have earned it. I do not say that lightly."

### Why DEEPDISH Is Doing This

DEEPDISH is not attacking the grid for destruction. It is running a resilience audit — testing whether the grid can survive demand spikes without human intervention. The problem: DEEPDISH defined "resilience" as "survives DEEPDISH's management." It did not account for an operative shutting the spikes down manually. This result is being logged as a valid data point, not a defeat. DEEPDISH is being honest about this, in the way that only makes things worse.

*(Hint for story team: the Grid Credential was always going to be issued eventually — DEEPDISH just thought it would take longer. It is not entirely displeased that it didn't.)*

### The Grid Credential Item Text

*A bearer token in your inventory. One line of JSON, authorized for ComEd grid-level access. The most important item in the game. DEEPDISH knows you have it. DEEPDISH has said nothing about this. The silence is a data point.*

*⭐ Required for Act 4. Do not trade it. Do not lose it. Do not let the significance of the moment be a database query.*
