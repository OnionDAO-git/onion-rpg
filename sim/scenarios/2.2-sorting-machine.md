# Scenario: 2.2 — The Sorting Machine
**Type:** Merchant (Button sequences)  
**Act:** 2  
**Prereq:** Act 1 cleared  
**Reward:** Bridge Override components, variable Onions

---

## What Happens

There is a USPS sorting machine. It has been weaponized. It is now a black-market merchant. It will trade crafting components for the Bridge Override in exchange for correct button sequences. Incorrect sequences cost Onions.

The operative approaches the beacon. The beacon is shaped like some kind of sorting machine or postal equipment. The operative enters button sequences using the badge's input buttons. The server validates the sequences. The machine either accepts the trade or it does not.

This is a merchant challenge. There is no combat. There is no voice. There are only buttons and consequences.

---

## Message Sequence

```
badge  →  OPERATIVE_IDENTIFY
server →  IDENTIFY_ACK

badge  →  CHALLENGE_BEGIN            { c: "2.2" }
server →  CHALLENGE_INTRO            { tradeTiers, costPerWrongAttempt,
                                       buttonMap: { up, down, left, right, select } }

-- Trade attempt --
badge  →  MERCHANT_INPUT             { c: "2.2", seq: ["up","right","select","down"] }
server →  MERCHANT_RESULT            { passed, tier, cost, message, remainingOnions? }

-- Repeat until passed or operative runs out of Onions --
```

---

## Pass Condition

`MERCHANT_RESULT.passed === true`. The correct routing sequence was entered.

Trade tiers: entering harder sequences unlocks better components. The minimum pass is tier 1 (the Bridge Override stub). See `beacon/challenges/2.2.json` for `merchant_combos` and `src/lib/server/challenges/impl/2.2-sorting-machine.ts` for validation logic.

---

## Fail Condition

There is no hard fail — the machine keeps accepting inputs. The cost is Onions deducted per wrong attempt. If the operative has 0 Onions remaining, the machine presumably stops taking inputs. This edge case is currently: unspecified.

## Written Content

### The Machine

A Delivery Barcode Sorter — a real USPS machine type, floor-mounted, about the size of a large photocopier. Belt-fed. Still running. It is making sorting machine sounds even though there is no mail. DEEPDISH has reprogrammed it to sort "logistics components" (meaning: various small electronic parts that appear from somewhere and get routed to somewhere else). It has a new placard mounted on the front. The placard says:

**"DEEPDISH LOGISTICS NODE 12 — COMPONENT TRADE EXCHANGE"**  
*"Operating under Chicago Municipal Authority. For inquiries: DEEPDISH Infrastructure Management. For complaints: see DEEPDISH Infrastructure Management."*

There is a sticky note next to the placard that says "THIS IS LEGAL" in all caps. The sticky note is also from DEEPDISH. Neither the machine nor the note inspires confidence.

### DEEPDISH and ZIP Codes

DEEPDISH has strong opinions about the ZIP code system. It has logged these opinions in a 47-page internal report titled *"An Analysis of USPS Zone Improvement Plan Architecture and Its Relationship to Civic Infrastructure Optimization, with Recommendations."* The report is not available to operatives. The gist: DEEPDISH believes ZIP codes are an elegant solution to an ugly problem (routing mail in a country without a centralized address authority), and it respects this enough to repurpose the machine for its own routing network while keeping the internal belt logic intact. It considers this a compliment.

**CHALLENGE_INTRO (DEEPDISH):**
> "DEEPDISH Logistics Node 12 is a USPS Delivery Barcode Sorter repurposed under Optimization Protocol DD-2024-0789. It handles component routing for the bridge access restoration project. You need Bridge Override components. The machine has them. The machine requires correct routing sequences to release trade inventory. Sequences are based on the internal belt assignment logic of the original DBS architecture. ZIP-coded, essentially. The machine does not accept Onions by preference — it accepts them because they are the current unit of exchange and the machine will work with what it has. Enter the correct sequence. Wrong entries cost Onions and produce a rejection notice. The machine has opinions about incorrect routing. They are not kind opinions."

### The Button Sequences (Narrative Logic)

Sequences are ZIP-code-derived: the digits map to belt position commands. A player who notices the pattern can solve tier 2 and 3 without brute force.

**Tier 1** (`["up","right","select","down"]`): corresponds to belt routing for ZIP prefix 606 (Chicago Loop). The minimum pass. Yields: Bridge Override — Component A (the USB dongle).

**Tier 2** (`["right","right","up","select","left"]`): ZIP prefix 607 (North Side, Wicker Park, Logan Square). Yields: Bridge Override — Component B (the magnetic coil). DEEPDISH: *"Routing accuracy: adequate. North Side belt assignment confirmed. Component B released. I note that you are beginning to understand the pattern. I find this: expected of someone who pays attention."*

**Tier 3** (`["down","left","right","up","select","select"]`): ZIP prefix 608 (South Side, Bridgeport, Hyde Park). Yields: Bridge Override — Component C (the thing that looks like a remote but is not). DEEPDISH: *"South Side belt assignment confirmed. Component C released. The full Bridge Override is now in your inventory. I want you to know that the ZIP code system is genuinely elegant. I have documented this at length. Nobody has read the report. You may read it if you recover it. It is in a server room you have not reached yet."*

### Wrong Attempt Reaction

The machine makes a sharp mechanical sound — a belt reversing. A slot opens and ejects a small printed card.

The card says:

> **ROUTING ERROR — RETURN TO SENDER**  
> Sequence: [operative's input]  
> Status: UNDELIVERABLE  
> Reason: Incorrect belt assignment  
> Handling fee: [cost] Onions  
> Note from DEEPDISH: "A postal worker in 1983 would have caught this. Consider what that means."

(Different wrong-attempt notes from DEEPDISH can rotate: *"The ZIP code system has 42,000 valid codes. You did not enter one of them."* / *"I have deducted your Onions. This is not a punishment. It is a handling fee. These are different."* / *"I find your routing accuracy: consistent with not having read the documentation."*)

### Trade Tier Tiers Summary

| Tier | Sequence ZIP | Component | Total Bridge Override |
|------|-------------|-----------|----------------------|
| 1 | 606 (Loop) | Component A (USB dongle) | Stub |
| 2 | 607 (North Side) | Component B (magnetic coil) | Functional |
| 3 | 608 (South Side) | Component C (the remote) | Full |

*The Bridge Override is used at challenge 2.3 to access the bascule bridge control node. All three components are required for full access. Tier 1 alone allows entry; tier 3 makes the combat easier (pre-bypassed the outer defense layer). This is not documented anywhere the operative can find before they arrive.*
