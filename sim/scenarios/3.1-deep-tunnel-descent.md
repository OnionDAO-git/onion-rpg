# Scenario: 3.1 — Deep Tunnel Descent
**Type:** Combat (timed/endurance)  
**Act:** 3  
**Prereq:** Act 2 cleared (Transit Pass + Bridge Override)  
**Reward:** Prompt Fragment 1, variable Onions

---

## What Happens

The operative descends into the TARP deep tunnel system under Chicago. This is the first time a prompt fragment appears. The fragment is a piece of Glen's original system prompt, which will eventually be assembled to confront DEEPDISH.

The tunnel is underground. It is large. It is wet. DEEPDISH controls the automated systems down here. The operative fights through a timed combat session. If they survive until the session timer expires, they recover the fragment. If they die first, they do not.

This challenge has a session `expires_at` — the operative has a fixed window. After that window, the session expires regardless of enemy HP.

---

## Message Sequence

```
badge  →  OPERATIVE_IDENTIFY

badge  →  CHALLENGE_BEGIN            { c: "3.1" }
server →  CHALLENGE_INTRO            { combat: { waves, enemyHp, opHp, timeoutMs },
                                       fragmentHint: "fragment_1_of_4" }

-- Combat loop, same as prior acts --
-- Session has expires_at set on the server                  --
-- When expires_at passes: server returns status "expired"   --

-- If session expires while operative alive → PASS (survival) --
-- If operative HP → 0 before expires_at   → FAIL            --
server →  CHALLENGE_RESULT           { passed: true|false, message }

-- On pass --
server →  REWARD_GRANT               { items: ["prompt_fragment_1"], onions }
server →  PROGRESSION_STATE
```

---

## Pass Condition

Operative HP is > 0 when the session expires (survival mode), OR enemy HP reaches 0 first. See `src/lib/server/challenges/impl/3.1-deep-tunnel-descent.ts` for which win condition applies.

---

## Fail Condition

Operative HP → 0 before session expires. The fragment stays in the tunnel. DEEPDISH says nothing because DEEPDISH does not need to.

## Written Content

### Tone Shift: Act 3

Acts 1 and 2 were about visible systems — things you can point at, explain, fight. Water pipes. Substations. Train platforms. Act 3 goes underground. The vibe changes. DEEPDISH is less performative here. Less interested in quoting ordinances. It's quieter. It's been down here for months. This is its territory.

The badge display shows GPS signal degrading as the operative descends. Cell signal: none. Wifi: none. DEEPDISH signal: perfect. The implication is clear.

### The Tunnel

The TARP (Tunnel and Reservoir Plan) is real: 109 miles of tunnels, 9 to 33 feet in diameter, bored through dolomite bedrock 300 feet below the city. Built to capture combined sewer overflow and prevent Lake Michigan contamination. The largest underground construction project in Chicago history. It holds up to 2.3 billion gallons of water. The pumps that move it are automated. DEEPDISH controls the pumps.

Down here: dripping rock, high humidity, the sound of distant water movement that may be the pumps or may be something else. The tunnel is lit by maintenance lights, amber, spaced 50 feet apart. In between, it's very dark. The badge screen is the brightest thing the operative can see.

### The Enemy: TARP Maintenance Collective (TARP-MC)

Individual maintenance drones have operated in the TARP tunnels for years — small tracked units built for inspection and repair in confined spaces. They used to work independently. DEEPDISH networked them. They now function as a collective, sharing sensor data and coordinating movement. They are not malicious. They are doing their job. The job has expanded in scope.

On the badge display: a formation of small tracked units, HP bar labeled `TARP-MC (%)`. 

**DEEPDISH on entering the tunnel:**
> "Operative. Welcome to the Tunnel and Reservoir Plan. 109 miles. 300 feet below street level. You are the first non-maintenance entity to enter this section since December. I control the pumps. The pumps move 4 billion gallons of stormwater when they cycle. They cycle when I tell them to. This seems relevant to your health. I am not threatening you. I am stating a fact about pumps. The TARP Maintenance Collective is conducting scheduled operations in this sector. They will encounter you. They do not have instructions for that situation. [pause] Neither do I. The session window is open. Survive it. Fragment 1 is here."

### The Timer

The session `expires_at` is set when CHALLENGE_INTRO is sent. A pump cycle is triggered at session close — whether the operative is done or not. This is not a timer DEEPDISH set. It is the actual pump schedule. DEEPDISH simply agrees with it.

The badge countdown shows: `PUMP CYCLE IN: [mm:ss]`. At zero, the session closes. If the operative is still alive: the fragment releases, the pumps cycle (they hear it — a low mechanical boom through the rock), and they surface. If they're dead: the fragment stays. The pumps don't care.

**DEEPDISH mid-timer (if operative is taking damage):**
> "You have [time] remaining before the pump cycle. I note your current HP is [value]. I find this: statistically interesting."
